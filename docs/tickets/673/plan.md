# Issue #673: Excel Download for Magistrates Public and Standard Hearing Lists

## 1. Technical Approach

### Strategy

Generate an Excel (.xlsx) file for each `MAGISTRATES_PUBLIC_LIST` and `MAGISTRATES_STANDARD_LIST` artefact at publication time, alongside the existing PDF. Store the Excel file in blob storage under the key `${artefactId}.xlsx` in `CONTAINER.PUBLICATIONS`. Track whether an Excel file was generated by adding an `excelPath String?` column to the `Artefact` model in the Prisma schema, so the display page can conditionally show the download link.

### Architecture Decisions

**Storage approach — `excelPath` on `Artefact` (not blob-existence check)**

The artefact page needs to know whether an Excel file exists before rendering the download link. Options considered:

1. Try to download `${artefactId}.xlsx` from blob storage on every page request — adds latency and blob storage cost to every render.
2. Add `excelPath String?` to `Artefact` — populated during processing alongside `pdfPath`. Zero extra latency at render time; the page reads it from the DB query it is already making.

The DB column approach is cleaner and consistent with how `pdfPath` is tracked. The column is nullable so older artefacts without an Excel file render without errors.

**File format — `.xlsx` via ExcelJS**

ExcelJS 4.4.0 is already a dependency of `@hmcts/list-types-common`. Using it for output (as well as the existing input parsing) keeps the dependency surface flat. `.xlsx` is the format referenced in the acceptance criteria (column: "Download as a spreadsheet").

**Excel row model**

- `MAGISTRATES_PUBLIC_LIST`: one row per case (or application). The existing `MagistratesPublicListData` renderer already produces `case.defendant`, `case.caseUrn`, `case.offences`, `case.reportingRestriction` etc. by mutating the JSON in place. The Excel generator calls the same renderer and walks the mutated data.
- `MAGISTRATES_STANDARD_LIST`: one row per offence, with defendant-level fields repeated. The existing `renderMagistratesStandardListData` renderer returns a typed `RenderedMagistratesStandardListData` with a full hierarchy (`RenderedCourtRoom` → `RenderedSitting` → `RenderedHearing` → `RenderedOffence[]`). The Excel generator flattens this into rows.

**Download endpoint — `?format=excel` query param**

The existing `/api/flat-file/:artefactId/download` route in `libs/public-pages/src/routes/api/flat-file/[artefactId]/download.ts` currently serves only the artefact's primary flat file. Rather than changing the contract of that route, extend it to accept an optional `?format=excel` query param. When `format=excel` is present, the handler reads the `excelPath` from the DB (via a new query helper) and serves the blob from `CONTAINER.PUBLICATIONS`. If no Excel file was generated the handler returns 404.

**Notifications — `excelDownloadUrl` personalisation field**

The GOV.UK Notify client in `libs/notifications/src/govnotify/govnotify-client.ts` currently attaches a single PDF via `prepareUpload`. The Notify template already has a `link_to_file` placeholder. Adding a second file attachment to the same email requires a new template placeholder and a new Notify template version — this is an out-of-band change to the Notify dashboard. The implementation adds an `excelDownloadUrl` plain-URL field to `TemplateParameters` and populates it in the notification path for these two list types, with documentation noting the template change needed.

**CSV injection prevention**

Column cell values that start with `=`, `+`, `-`, or `@` are prefixed with a single-quote `'` before writing to the workbook. This is applied in a shared sanitisation helper.

**Content type**

Add `.xlsx` → `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` to `libs/publication/src/file-storage/content-type.ts`.

### Key Constraints

- `MAGISTRATES_STANDARD_LIST` and `MAGISTRATES_PUBLIC_LIST` are already in `PDF_GENERATOR_REGISTRY` in `libs/publication/src/processing/service.ts`. The new Excel generation runs in `processPublication` after the PDF step.
- The `Artefact.type` field exists in the real schema (`base.prisma`) but is absent from the generated Prisma snapshot. Do not rely on it; use `listTypeName` from the `listType` relation join.
- `getArtefactById` already performs the `listType` join and returns `ArtefactWithListType` with a non-null `listTypeName`. All routing on list type uses this string name, never the numeric `listTypeId`.
- The `deleteArtefacts` function in `libs/publication/src/repository/queries.ts` must be extended to also delete `${artefactId}.xlsx` from `CONTAINER.PUBLICATIONS` when cleaning up.

### 2.5 Magistrates Public List Excel generator

**File:** `libs/list-types/magistrates-public-list/src/excel/excel-generator.ts` (new)

Columns (in order): Court House, Court Room, Sitting at, URN, Name, Hearing Type, Prosecuting Authority, Offence Details, Reporting Restrictions

Logic:
1. Call `renderMagistratesPublicListData(jsonData, options)` to get the mutated data.
2. Create an ExcelJS workbook with one worksheet named after the list type friendly name.
3. Write a header row using column header strings from the locale file matching the `locale` option (`en` or `cy`).
4. Walk `listData.courtLists → courtHouse → courtRoom → session → sittings → hearing → case[]` and `application[]`, writing one row per case/application. Where a case has multiple offences, join their details as a comma-separated string in the Offence Details cell.
5. Sanitise all string cell values via `sanitiseCellValue`.
6. Write the workbook to a buffer (`workbook.xlsx.writeBuffer()`).
7. Call `saveExcelToStorage(artefactId, buffer)`.

Interface:
```typescript
interface ExcelGenerationOptions {
  artefactId: string;
  locationId: string;
  contentDate: Date;
  locale: string;
  jsonData: MagistratesPublicListData;
}

interface ExcelGenerationResult {
  success: boolean;
  excelPath?: string;
  error?: string;
}

export async function generateMagistratesPublicListExcel(options: ExcelGenerationOptions): Promise<ExcelGenerationResult>
```

Add `exceljs: "4.4.0"` to `libs/list-types/magistrates-public-list/package.json` dependencies.

Export `generateMagistratesPublicListExcel` from `libs/list-types/magistrates-public-list/src/index.ts`.

### 2.6 Magistrates Standard List Excel generator

**File:** `libs/list-types/magistrates-standard-list/src/excel/excel-generator.ts` (new)

Columns (in order per acceptance criteria):
Court House, LJA, Court Room, Sitting at, Name, Application Particulars, DOB, Age, Address, Prosecuting Authority Name, Attendance Method, Reference, Application Type, ASN, Hearing Type, Panel, Reporting Restrictions, Offence Code, Offence Title, Offence Details, Legislation, Max Penalty, Plea, Date of Plea, Convicted on, Adjourned from

Logic:
1. Call `renderMagistratesStandardListData(jsonData, options)` to get `RenderedMagistratesStandardListData`.
2. Create workbook with one worksheet.
3. Write header row.
4. Walk `listData → RenderedCourtRoom → RenderedSitting → RenderedHearing → RenderedOffence[]`. For each offence on a hearing, write one row repeating the court/defendant-level fields.
5. If a hearing has zero offences, write one row with empty offence columns.
6. Sanitise all string values.
7. Save to storage.

**File:** `libs/public-pages/src/flat-file/flat-file-service.ts`

Add `getExcelForDownload(artefactId: string)`:
- Call `getArtefactById(artefactId)` — validates existence, expiry.

- Download the blob from `CONTAINER.PUBLICATIONS` using `downloadBlob(artefact.excelPath, CONTAINER.PUBLICATIONS)`.

## 4. Error Handling and Edge Cases

**Excel generation fails during processing**
Excel errors are non-fatal. `generatePublicationExcel` catches all exceptions, logs them, and returns `{}` (no `excelPath`). The PDF path is unaffected. `artefact.excelPath` remains null. The display page shows only the PDF download link.

**Artefact published before this feature (no `excelPath`)**
`excelPath` is a nullable column. `getArtefactById` returns `excelPath: null` for older artefacts. The display page omits the Excel link. The download route returns 404 if `format=excel` is requested for such an artefact.

**`deleteArtefacts` — Excel blob missing in storage**
The existing pattern for PDF deletion suppresses 404 errors from blob storage. The same pattern is applied to the new `.xlsx` deletion.

**`format` query param with invalid value**
Unknown values default silently to `"pdf"`. This prevents enumeration of supported formats being used to probe storage. The allow-list (`pdf`, `excel`) is enforced server-side only; the format value is never used to construct a blob key directly.

**CSV injection**
Any cell value beginning with `=`, `+`, `-`, or `@` is prefixed with `'` by `sanitiseCellValue` before writing to the worksheet. This is applied to all string cell values in both generators.

**Very large lists**
ExcelJS builds the workbook in memory. For Magistrates lists the row count is bounded by the number of hearings in a day; memory impact is not expected to be significant. If it becomes an issue, ExcelJS streaming writes can be adopted without changing the public interface.
